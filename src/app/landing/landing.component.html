Features

Focus on writing use cases

Tests are written in simple commands. Your testers should not be focusing on how to test something, but rather on what to test. Behind the scenes we take care of translating it to robust code

Beyond UI testing

Set up connectors which can make your tests interact with complex internal systems, not just UI. Ensure records show up in a database, hit internal REST APIs, see if that mail really arrived.

Run it centrally but securely

Download and place the PROXY inside your network. Without any additional configuration, it will connect to the central platform and run any tests you send its way. It is self-updating - so you don't even have to worry about maintaining it.
